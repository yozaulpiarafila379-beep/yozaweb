Object Oriented Programming (OOP)

01. Pengertian OOP

Object Oriented Programming (OOP) adalah paradigma pemrograman yang berfokus pada objek sebagai unit utama. Setiap objek memiliki state (data) dan behavior (perilaku). Tujuan OOP adalah membuat kode yang modular, reusable, dan mudah dipelihara dalam sistem yang kompleks.

02. Konsep Dasar

Class : Blueprint atau cetakan untuk membuat objek.
Object : Instance nyata yang tercipta dari sebuah class.
Attribute : Representasi data melalui variabel di dalam class.
Method : Logika fungsional atau fungsi di dalam class.

03. Class Sederhana (Car)

Class adalah blueprint untuk membuat objek. Pada contoh ini, class Car memiliki atribut model dan color serta method start() dan stop().

class Car {
  String? model;
  String? color;

  void start() {
    print('Car started!');
  }

  void stop() {
    print('Car stopped!');
  }
}

void main() {
  Car myCar = Car();
  
  myCar.model = "Sedan";
  myCar.color = "Merah";

  myCar.start();
  myCar.stop();
}

06. Encapsulation (Person)

Teknik keamanan data dengan menyembunyikan akses langsung ke variabel menggunakan prefix underscore, diakses melalui metode getter dan setter.

DART SOURCE CODE

// Nama        : Yoza Ulpia Rafila
// Nim         : 251410004
// Kelas       : SI2A

class Person {
  String _name;
  int age;

  Person(this._name, this.age);

  void setName(String name) => _name = name;
  String getName() => _name;
}

void main() {
  Person person = Person("Yoza Ulpia Rafila", 19);

  print(person.age);
  person.setName("Yoza Ulpia Rafila");
  print(person.getName());
}

08. Named Parameters (Produk)

Penggunaan named parameters dan required untuk membuat objek yang lebih deskriptif, dengan nilai default pada beberapa atribut.

DART SOURCE CODE

// Nama        : Yoza Ulpia Rafila
// Nim         : 251410004
// Kelas       : SI2A

// Membuat class Produk
// Class adalah blueprint atau rancangan untuk membuat objek produk
class Produk {
  // Deklarasi atribut / properti
  // final artinya nilainya tidak bisa diubah setelah objek dibuat
  final String id, nama;
  final double harga;
  final int stok;
  final String kategori;

  // Constructor dengan named parameter
  // required artinya wajib diisi saat membuat objek
  // stok dan kategori memiliki nilai default
  Produk({
    required this.id,
    required this.nama,
    required this.harga,
    this.stok = 0,
    this.kategori = 'Umum',
  });
}

void main() {
  // Membuat objek pertama bernama laptop
  // Semua atribut diisi lengkap
  Produk laptop = Produk(
    id: "P001",
    nama: "Laptop Gaming",
    harga: 15000000,
    stok: 15,
    kategori: "Elektronik",
  );

  // Membuat objek kedua bernama smartphone
  // kategori tidak diisi → otomatis bernilai "Umum"
  Produk smartphone = Produk(
    id: "P002",
    nama: "Smartphone Flagship",
    harga: 12000000,
    stok: 25,
  );

  // Membuat objek ketiga bernama kamera
  // stok dan kategori tidak diisi
  // stok otomatis = 0
  // kategori otomatis = "Umum"
  Produk kamera = Produk(
    id: "P003",
    nama: "Kamera Mirrorless",
    harga: 20000000,
  );

  // Menampilkan nama masing-masing objek ke console
  print(laptop.nama); // Output: Laptop Gaming
  print(smartphone.nama); // Output: Smartphone Flagship
  print(kamera.nama); // Output: Kamera Mirrorless
}

09. Kesimpulan

Dari berbagai contoh kode yang telah ditampilkan, dapat disimpulkan bahwa Object Oriented Programming (OOP) memudahkan pengelolaan kode dengan mengelompokkan data dan perilaku ke dalam kelas. Setiap kelas dapat memiliki atribut untuk menyimpan data dan method untuk menjalankan aksi.

Konsep seperti konstruktor (posisional dan bernama) membantu dalam membuat objek dengan cara yang lebih fleksibel, sedangkan properti tersembunyi membantu dalam melindungi data agar tidak diakses secara sembarangan. Dengan menerapkan OOP, kode yang dibuat akan menjadi lebih terstruktur, mudah diperbaiki, dan dapat digunakan kembali untuk kebutuhan yang berbeda.

10. Quiz Bonus Kambing Class

Class adalah blueprint untuk membuat objek. Pada contoh ini, class Hewan memiliki atribut nama dan method bersuara(), sedangkan class QuizBonusKambing memiliki atribut jenisKambing dan method tampilkanJenis() untuk keperluan quiz bonus.

// Nama : Yoza Ulpia Rafila

// Kelas : SI2A

class Hewan{
  String nama;

  Hewan(this.nama);

  void bersuara() {
    print('$nama sedang bersuara...');
    print('Suara: Mbeeek...');
  }
}

// Kelas Quiz Bonus Kambing
class QuizBonusKambing {
  // Contoh properti dan metode bisa ditambahkan sesuai kebutuhan

  var myPet = Hewan('Kambing');
