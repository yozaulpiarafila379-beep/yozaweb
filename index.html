Object Oriented Programming (OOP) dalam Dart

Pengertian Umum OOP: Paradigma pemrograman yang berfokus pada objek sebagai unit utama. Setiap objek memiliki state (data/atribut) dan behavior (perilaku/metode). Tujuan OOP adalah membuat kode modular, reusable, dan mudah dipelihara untuk sistem kompleks.

Contoh Kode 1: Class Car dengan Metode Start & Stop
class Car {
  String? model;
  String? color;

  void start() {
    print('Car started!');
  }

  void stop() {
    print('Car stopped!');
  }
}

void main() {
  Car myCar = Car();
  myCar.model = "Sedan";
  myCar.color = "Merah";

  myCar.start();
  myCar.stop();
}

Pengertian:

  • class Car: Mendefinisikan kelas Car sebagai cetakan untuk membuat objek mobil. Kelas berisi atribut dan metode yang menjadi ciri objek tersebut.
  • String? model; String? color;: Atribut (state) dari kelas Car dengan tipe data String? (boleh bernilai null) untuk menyimpan model dan warna mobil.
  • void start() & void stop(): Metode (behavior) yang mendefinisikan perilaku mobil, yaitu menghidupkan dan mematikan mesin dengan menampilkan pesan ke konsol.
  • Car myCar = Car();: Membuat objek myCar dari kelas Car menggunakan konstruktor default.
  • myCar.model = "Sedan";: Mengisi nilai atribut objek setelah dibuat.
  • myCar.start();: Memanggil metode start() untuk menjalankan perilaku objek.
Contoh Kode 2: Class Car dengan Menampilkan Properti
class Car {
  String? model;
  String? color;
}

void main() {
  Car myCar = Car();
  myCar.model = "Sedan";
  myCar.color = "Merah";

  print(myCar.model);
  print(myCar.color);
}

Pengertian:

  • Kelas Car hanya memiliki atribut tanpa metode tambahan.
  • print(myCar.model);: Menampilkan nilai atribut (properti) objek myCar ke konsol. Hal ini berguna untuk melihat atau mengakses data yang disimpan dalam objek.
  • Output yang dihasilkan: Sedan dan Merah.
Contoh Kode 3: Class Car dengan Konstruktor Posisional
class Car {
  String model;
  String color;

  Car(this.model, this.color);

  void start() {
    print('Engine Online: $model');
  }
}

void main() {
  Car myCar = Car("Sedan", "Merah");
  myCar.start();
  print(myCar.color);
}

Pengertian:

  • String model; String color;: Atribut tidak boleh bernilai null (tanpa tanda ?), sehingga perlu diinisialisasi saat objek dibuat.
  • Car(this.model, this.color);: Konstruktor posisional yang digunakan untuk mengisi nilai atribut secara langsung saat pembuatan objek. Urutan parameter harus sesuai dengan deklarasi konstruktor.
  • print('Engine Online: $model');: Menggunakan $model untuk menyisipkan nilai atribut ke dalam string (string interpolation).
  • Car myCar = Car("Sedan", "Merah");: Membuat objek dan mengisi nilai atribut melalui konstruktor posisional.
Contoh Kode 4: Class Mahasiswa
class Mahasiswa {
  String nama;
  String nim;
  double ipk;

  Mahasiswa(this.nama, this.nim, this.ipk);

  void tampilkanData() {
    print('Nama : $nama');
    print('NIM : $nim');
    print('IPK : $ipk');
  }
}

void main() {
  Mahasiswa mhs1 = Mahasiswa("Budi Santoso", "20230001", 3.75);
  mhs1.tampilkanData();
}

Pengertian:

  • Kelas Mahasiswa dibuat untuk merepresentasikan data dan perilaku mahasiswa.
  • Atribut nama (String), nim (String), dan ipk (double) menyimpan data mahasiswa.
  • Mahasiswa(this.nama, this.nim, this.ipk);: Konstruktor posisional untuk menginisialisasi semua atribut saat objek dibuat.
  • void tampilkanData(): Metode yang mengelompokkan perintah untuk menampilkan semua data mahasiswa ke konsol, sehingga kode lebih terstruktur dan reusable.
  • Output yang dihasilkan: Data lengkap mahasiswa Budi Santoso.
Contoh Kode 5: Class Person dengan Properti Tersembunyi
class Person {
  String _name;
  int age;

  Person(this._name, this.age);

  void setName(String name) => _name = name;
  String getName() => _name;
}

void main() {
  Person person = Person("Yoza Ulpia Rafila", 19);
  print(person.age);
  person.setName("Yoza Ulpia Rafila");
  print(person.getName());
}

Pengertian:

  • String name;: Atribut dengan awalan merupakan properti tersembunyi (private), yang seharusnya tidak diakses langsung dari luar kelas. Ini adalah konsep enkapsulasi dalam OOP.
  • int age;: Atribut publik yang bisa diakses langsung dari luar kelas.
  • void setName(String name) => _name = name;: Metode setter untuk mengubah nilai atribut _name secara terkontrol.
  • String getName() => _name;: Metode getter untuk mengambil nilai atribut _name.
  • Enkapsulasi bertujuan melindungi data agar tidak diubah secara sembarangan, menjaga konsistensi nilai.
Contoh Kode 6: Class Product dengan Konstruktor Sederhana
class Product {
  String nama;

  Product(this.nama);
}

void main() {
  Product produk1 = Product("Laptop Gaming");
  print(produk1.nama);
}

Pengertian:

  • Kelas Product dengan atribut tunggal nama.
  • Product(this.nama);: Konstruktor sederhana dengan satu parameter posisional untuk menginisialisasi nama produk.
  • Product produk1 = Product("Laptop Gaming");: Membuat objek produk dan menetapkan nama secara langsung melalui konstruktor.
  • Output: Laptop Gaming.
Contoh Kode 7: Class Produk dengan Konstruktor Bernama
class Produk {
  final String id, nama;
  final double harga;
  final int stok;
  final String kategori;

  Produk({
    required this.id,
    required this.nama,
    required this.harga,
    this.stok = 0,
    this.kategori = "Umum",
  });
}

void main() {
  Produk laptop = Produk(
    id: "P001",
    nama: "Laptop Gaming",
    harga: 15000000,
    stok: 15,
    kategori: "Elektronik",
  );

  Produk smartphone = Produk(
    id: "P002",
    nama: "Smartphone Flagship",
    harga: 12000000,
    stok: 25,
  );
}

Pengertian:

  • final: Menandakan bahwa nilai atribut tidak bisa diubah setelah diinisialisasi, menjaga immutabilitas data.
  • Produk({ ... }): Konstruktor bernama (named parameters) yang memudahkan pembacaan kode karena parameter disebutkan dengan namanya saat objek dibuat.
  • required: Menandakan parameter yang wajib diisi saat membuat objek.
  • this.stok = 0; this.kategori = "Umum": Parameter opsional dengan nilai default, sehingga tidak perlu diisi jika tidak ada perubahan.
  • Objek smartphone tidak mengisi parameter kategori, sehingga akan menggunakan nilai default "Umum".
