OBJECT ORIENTED PROGRAMMING (OOP) DALAM DART

01 PENGERTIAN OOP

Object Oriented Programming (OOP) adalah paradigma pemrograman yang berfokus pada objek sebagai unit utama. Setiap objek memiliki state (data) dan behavior (perilaku). Tujuan OOP adalah membuat kode yang modular, reusable, dan mudah dipelihara dalam arsitektur sistem yang kompleks.

02 KONSEP DASAR

  • Class: Blueprint atau cetakan protokol untuk membuat objek.
  • Object: Manifestasi runtime atau instance nyata dari sebuah class.
  • Attribute: Representasi data melalui variabel di dalam class.
  • Method: Logika fungsional atau fungsi di dalam class.

Empat pilar utama ini membentuk struktur dasar dalam pengembangan aplikasi berbasis objek di Dart, memungkinkan pengembang untuk memodelkan entitas dunia nyata ke dalam kode digital.

03 CONTOH CLASS SEDERHANA (CAR)

Di bawah ini adalah contoh pendefinisian class Car. Class ini memiliki dua attribute (model dan color) serta dua method (start dan stop) yang merepresentasikan perilaku dasar sebuah mobil.

class Car {
  String? model;
  String? color;

  void start() {
    print('Car started!');
  }

  void stop() {
    print('Car stopped!');
  }
}

04 OBJECT INSTANTIATION

Setelah class didefinisikan, kita dapat membuat Object atau instance dari class tersebut di dalam fungsi main(). Proses ini disebut instansiasi, di mana objek dialokasikan ke dalam memori.

void main() {
  Car myCar = Car();
  myCar.model = "Sedan";
  myCar.color = "Merah";

  myCar.start();
  myCar.stop();
}

05 CONSTRUCTORS

Constructor adalah metode inisialisasi yang dipanggil secara otomatis saat objek baru diciptakan dalam memori sistem. Di Dart, kita bisa menggunakan formal parameters (this.attribute) untuk menyederhanakan proses pengisian nilai attribute.

class Car {
  String model;
  String color;

  Car(this.model, this.color);

  void start() {
    print('Engine Online: $model');
  }
}

06 PRACTICE: MAHASISWA CLASS

Latihan dalam membuat class Mahasiswa yang menampung data akademik. Class ini mengimplementasikan constructor untuk memastikan setiap objek mahasiswa memiliki data yang lengkap saat dibuat.

class Mahasiswa {
  String nama;
  String nim;
  double ipk;

  Mahasiswa(this.nama, this.nim, this.ipk);

  void tampilkanData() {
    print('NAMA : $nama | NIM : $nim | IPK : $ipk');
  }
}

void main() {
  Mahasiswa mhs1 = Mahasiswa("Yoza Ulpia Rafila", "251410004", 3.9);
  mhs1.tampilkanData();
}

07 ENCAPSULATION

Encapsulation adalah teknik untuk menyembunyikan detail implementasi internal. Di Dart, kita menggunakan awalan underscore (_) untuk membuat attribute bersifat private, sehingga hanya bisa diakses melalui getter dan setter.

class Person {
  String _name;
  int age;

  Person(this._name, this.age);

  void setName(String name) => _name = name;
  String getName() => _name;
}

08 THIS KEYWORD

Kata kunci this digunakan untuk merujuk pada instance saat ini dari sebuah class. Ini sangat berguna untuk menghindari ambiguitas ketika nama parameter pada constructor sama dengan nama attribute di dalam class.

class Product {
  String nama;
  Product(this.nama);
}

09 PRACTICE 2: PRODUK CLASS

Latihan lanjutan menggunakan Named Parameters dan operator required. Teknik ini membuat pembuatan objek menjadi lebih deskriptif dan aman karena compiler akan memberikan peringatan jika data wajib tidak diisi.

class Produk {
  final String id, nama;
  final double harga;
  final int stok;
  final String kategori;

  Produk({
    required this.id,
    required this.nama,
    required this.harga,
    this.stok = 0,
    this.kategori = 'Umum',
  });
}

10 KESIMPULAN

  • OOP mempermudah pengelolaan kode melalui enkapsulasi logika dalam Object.
  • Constructor digunakan untuk inisialisasi data primer secara otomatis.
  • Encapsulation menjaga keamanan data dari akses runtime ilegal.
  • Dart mendukung OOP secara native, menjadi fondasi utama framework Flutter.
